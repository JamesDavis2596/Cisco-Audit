Security

The security at ACME is something that needs to be worked on in a more aggressive manner as there are no firewalls in place throughout any of the infrastructure which will need to be addressed at the earliest as the lack of security features in place is leaving the company at a vulnerable state for someone to attack.

Following on from this, there seems to be a lack of secure passwords on all of the systems as all are using a simple passphrase like “Cisco123” or “Branch4”. This is seen across all desktop PC’s and network access points. This is leaving the network vulnerable to a brute force/dictionary attack, for someone to then have unlimited access across all systems in the network.

A security standard should be set in place to change the passwords every 90 days and this can be forced out by using the domain commands by a domain admin.

All guest devices currently have access to the whole network without any security measures in place. This is another direct connection into the company's network that will need to be addressed as soon as possible.

Architecture

The infrastructure has almost all connections to everything in the network connected via Copper Straight-Through cabling whereas the “Branch Desktop Switch” is using copper crossover cabling and all the access points have access for devices to connect via Wi-Fi.

There is currently a lack of connection between the “HQ Master Switch” and the “HQ Desktop Switch”. This should be addressed to allow access between the HQ PC’s and the main servers in the network.
